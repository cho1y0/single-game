## 5. 네트워크 통신 및 예외 처리 (Network & Error Handling)
싱글 플레이어 게임이더라도 클라이언트(유니티)와 서버(톰캣)가 API로 통신하는 구조이므로, 네트워크 환경에 대한 대비가 필요합니다.

* **API 엔드포인트 변수화:** 유니티 코드 내에 `http://localhost:8080/api/mix` 처럼 주소를 직접 적지 마세요. 나중에 로컬 환경이 아닌 네이버 클라우드 같은 외부 서버 환경으로 백엔드를 이관하게 될 경우를 대비해, 최상위 설정 파일에 `BASE_URL`을 빼두고 참조하는 방식으로 구현해야 합니다.
* **서버 다운 및 타임아웃 처리:** 톰캣 서버가 꺼져 있거나 MySQL 데이터베이스 연결이 끊어졌을 때, 유니티 게임 화면이 무한 로딩에 빠지거나 멈추지 않도록 해야 합니다. API 호출 시 반드시 예외 처리(Try-Catch)를 하고, 실패 시 유저에게 "서버와의 연결이 원활하지 않습니다"라는 UI 팝업을 띄워야 합니다.
* **식별자(Session/Token) 관리:** 혼자 하는 싱글 게임이라도 API 통신 시 "내가 누구인지"를 서버에 알려주어야 내 인벤토리 DB를 조회할 수 있습니다. 로그인 시 발급받은 `user_id`나 임시 세션 값을 클라이언트 메모리에 들고 있다가, API 요청 시 항상 함께 전송해야 합니다.
