## 6. 로깅 및 디버깅 (Logging & Debugging)
버그가 발생했을 때 프론트엔드의 문제인지, 백엔드의 문제인지 빠르게 파악하기 위한 장치입니다.

* **백엔드 로그:** JSP나 Java 로직 처리 시, 톰캣 서버 콘솔에 중요 동작들을 기록(Log)하세요. (예: `[INFO] User 1 - Mix Attempt: Material(101, 102) -> Success!`) 에러가 났을 때 로그를 보면 쿼리문이 틀렸는지, 데이터가 안 넘어왔는지 즉각 파악할 수 있습니다.
* **유니티 디버그 로그:** 유니티 쪽에서도 서버로 어떤 데이터를 보냈고(`Debug.Log`), 어떤 JSON 응답을 받았는지 콘솔에 찍어두면 통신 에러를 잡기 훨씬 수월해집니다.
