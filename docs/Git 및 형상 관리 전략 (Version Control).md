## 3. Git 및 형상 관리 전략 (Version Control)
유니티 프로젝트는 용량이 크고, 여러 파일이 동시에 변경되므로 충돌 관리가 중요합니다.

* **.gitignore 필수 세팅:** 유니티의 `Library/`, `Temp/`, `Logs/` 폴더 및 서버의 컴파일된 `.class` 파일 등은 Git에 올라가지 않도록 철저히 필터링합니다. (GitHub에서 제공하는 Unity 및 Java 기본 템플릿 사용 권장).
* **브랜치(Branch) 전략:**
    * `main`: 항상 실행 가능한 안정적인 최종 배포 상태를 유지합니다.
    * `feat/unity-map`: 유니티 맵 제작 및 프론트 기능 작업 전용.
    * `feat/api-mix`: 백엔드 믹스 로직, DB 연동 및 API 구현 전용.
* **커밋(Commit) 원칙:** "프론트/백엔드 동시 수정" 보다는 "플레이어 이동 구현", "믹스 API 생성" 등 명확한 기능 단위로 쪼개어 커밋 기록을 남깁니다.
