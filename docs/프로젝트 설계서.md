# **📜 믹스마스터 싱글 프로젝트 상세 설계서 (GDD v3.0)**

## **1\. 게임 개요 및 세계관**

* **프로젝트명:** 믹스마스터 싱글 클론 (로컬 구동용)  
* **장르:** 2D 탑뷰/쿼터뷰 몬스터 수집형 RPG  
* **핵심 루프:** 마을(정비/NPC) ➡️ 필드(사냥/코어 획득) ➡️ 믹스(성장/마스터리) ➡️ 상위 필드 진출

### **1.1. 맵 구조**

* **마을 (안전 지대):** \* 메크리타, 헤르세바 등 원작의 주요 마을 구현.  
  * 몬스터가 스폰되지 않으며, 플레이어의 이동 및 NPC 상호작용만 가능.  
* **필드 (전투 지대):** \* 단계별(Tier) 필드 헨치가 서식하는 사냥터.  
  * 마을의 특정 포탈을 통해 이동하며, 입장 시 레벨/퀘스트 조건 체크.

---

## **2\. 기술 스택 및 아키텍처**

클라이언트와 서버를 분리하여 데이터의 무결성을 유지하는 구조를 채택합니다.

* **Frontend (클라이언트):** Unity 2D (C\#)  
* **Backend (서버/API):** Java, JSP (Tomcat 로컬 서버 구동)  
* **Database (데이터베이스):** MySQL  
* **버전 관리:** Git & GitHub (Repository 내 `client`, `server`, `database` 폴더 분리)

---

## **3\. 핵심 시스템 상세 로직**

### **3.1. 믹스(Mix) 및 마스터리(Mastery) 시스템**

게임의 가장 핵심이 되는 몬스터 합성 시스템입니다.

1. **믹스 단계:** 1단계(Tier 1)부터 최고 10단계(Tier 10)까지 존재.  
2. **기본 성공률:** 상위 단계일수록 기본 확률 하락 (예: 1단계 90% ➡️ 10단계 5%).  
3. **마스터리 누적:** 믹스 시도 시(성공/실패 무관) 해당 Tier의 마스터 포인트(`current_point`) 1 증가.  
4. **마스터 달성:** 누적 포인트가 목표치(`max_point`)에 도달하면 마스터 달성(`is_master = 1`).  
5. **마스터 보너스 (하위 ➡️ 상위):** \* 하위 단계를 마스터할수록 상위 단계 믹스 시 보너스 확률 부여.  
   * **최종 확률 공식:** `기본 확률 + (마스터한 하위 단계 개수 * 보너스 수치(%))`

### **3.2. NPC 상호작용 시스템**

마을 내 배치된 NPC와 상호작용(Space 바 또는 마우스 클릭)하여 기능을 수행합니다.

* **믹스 빌더:** 믹스 UI 호출 및 로직 수행.  
* **아이템 상인:** 소비 아이템(물약, 장비) 구매/판매 창 호출.  
* **안내원 / 기타:** 튜토리얼 대화 및 퀘스트 진행 상태 업데이트.

### **3.3. 전투 및 코어 드롭**

* **데미지 계산:** `(내 헨치 공격력 - 적 헨치 방어력)`을 기반으로 한 직관적인 사칙연산 적용 (최소 데미지 1).  
* **드롭 시스템:** 필드 헨치 처치 시, 난수(Random) 계산을 통해 정해진 확률로 '코어(Core)' 아이템을 인벤토리에 지급.

## **4\. 데이터베이스 스키마 설계 (MySQL)**

총 8개의 테이블로 구성하여 정적 데이터와 동적 데이터를 분리 관리합니다.

### **\[정적 데이터 (메타데이터)\]**

개발자가 사전에 세팅하는 시스템 데이터입니다.

* **`tb_map_info`**: 맵 기본 정보  
  * `map_id` (PK), `map_name` (예: 메크리타), `map_type` (마을/필드)  
* **`tb_npc_info`**: NPC 정보  
  * `npc_id` (PK), `map_id` (FK), `npc_name`, `role` (믹스/상점 등), `pos_x`, `pos_y`  
* **`tb_monster_dict`**: 몬스터 도감 및 스탯  
  * `dict_id` (PK), `name`, `tier` (1\~10), `base_hp`, `base_atk`  
* **`tb_mix_formula`**: 믹스 레시피  
  * `formula_id` (PK), `material_1` (FK), `material_2` (FK), `result_dict_id` (FK), `base_rate` (기본 확률)

### **\[동적 데이터 (유저 데이터)\]**

플레이어의 진행에 따라 실시간으로 변하는 데이터입니다.

* **`tb_user`**: 유저 상태  
  * `user_id` (PK), `nickname`, `gold`, `current_map_id` (FK), `pos_x`, `pos_y`  
* **`tb_user_hench`**: 보유 헨치 개별 정보  
  * `hench_uid` (PK), `user_id` (FK), `dict_id` (FK), `is_equipped` (0: 보관, 1: 동행)  
* **`tb_user_inventory`**: 소비 아이템 보관함  
  * `item_uid` (PK), `user_id` (FK), `item_id`, `quantity`  
* **`tb_mix_mastery`**: 유저별 믹스 숙련도  
  * `user_id` (FK), `tier` (1\~10), `current_point`, `max_point`, `is_master` (0/1)

## **5\. 주요 API 명세서 (백엔드 통신)**

클라이언트(Unity)가 톰캣(Tomcat) 서버로 요청하는 주요 기능입니다.

* `POST /api/user/move`: 주기적으로 플레이어의 현재 맵 ID와 좌표를 DB에 업데이트.  
* `GET /api/npc/interact`: 특정 NPC 클릭 시 해당 NPC의 고유 기능과 대화 스크립트 데이터 반환.  
* `POST /api/hench/mix`: 믹스 요청 처리 (확률 계산 로직 수행, 마스터리 포인트 업데이트, 인벤토리/헨치 DB 갱신).  
* `POST /api/battle/result`: 몬스터 처치 시 결과(코어 드롭 판정 및 골드 획득)를 백엔드에서 검증하고 저장.
